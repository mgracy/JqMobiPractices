Sench Touch's优劣

  • 可同时支持不同的device显示方式（Phone，Tablet，Windowsphone）
  • 强调one page one application 的开发方式，让你的web mobile app的操作方式更贴近于原生的app
  • 元件丰富
  • api文档完整
  • 较少的css来控制ui layout
  • 可转成android的原生码的可能性，可再制开发

  • touch基底的程序码较大约2m，初次加载时间比较长
  • 程序开发框架不易掌握
  • 不懂JS除错的人学习曲线将拉的很长
